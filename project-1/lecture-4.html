    Lecture 4

    Internet Protocols

    A protocol is a set of rules that devices aggree to follow so that they can
    communicate reliably (TCP/IP are examples of protocols)

    Originally, there were four layers. Today, there are seven layers in networking.

    Link Layer

    The lowest layer responsible for transmission and propagation of data.

    MAC (media access control) address is an identifier given to every
    NIC (network interface card).

    Internet Layer

    Routes traffic between communication partners acrosss networks.

    IP (Internet Protocol) a logical address that uniquely identify destinations
    on the Internet. Everything connected to the internet has an IP address

    Port Address Translation (PAT) is the reason we can still use ipv4 addresses,
    and not blow up the internet. A router turns any connected private ip address into
    one outward facing IP address which dramatically reduces the count
    of public IP addresses.

    Transport Layer

    Ensures transmissions arrive in order and without error.

    Transmission Control Protocol (TCP)

    • The most common transport protocol used in web browsing.
    • Each packet contains a header with a sequence number which allows ordered
      reconstrucion of the original message
    • Every packet acknowledges its successful arrival back to the sender (ACK)
    • If an ACK is not recieved in a given time, restransmit the packet
    • guaranteed that the message has been sent once you receive the ACK
    Application Layer

    Implement process-to-process communication

    1. HTTP(S)
    2. SSH
    3. FTP
    4. POP/IMAP/SMTP
    5. DNS

    Domain Name System

    Resolves domain names to IP addresses

    Humans better remember names than they do numbers. Therefore we use DNS as an
    abstraction to IP addresses

    Uniform Resource Locators (URLS)

    Let clients request particular resources (files) from the server

    1. the protocol used to connect
    2. the domain (or IP address) to connect to
