    Lecture 5

    HyperText Transfer Protocol

    Establishes a TCP connection on port 80 (by default). The server waits for the
    request, then responds with:

    • Headers
    • Response code
    • An optional message (which can include files)

    Request methods:

    • GET
    • POST
    • PUT
    • DELETE

    Less common methods:

    • CONNECT
    • TRACE
    • OPTIONS

    Web Browsers

    Parses HTML, CSS, and Javascript in order to render websites.

    Performance Metrics (Can get from Lighthouse↗):

    • Time to First Byte (TTFB)
    • First Paint (FP)
    • First Contentful Paint (FCP)
    • First Meaningful Paint (FMP)
    • Largest Contentful Paint (LCP)
    • Time to Interactive (TTI)
    • On load
    • Cumulative Layout Shift (CLS)

    Web Servers

    A web server is simply just a computer which provides services in
    different ways.

    • Backend API's
    • Databases
    • etc.

    What is HTML and where it comes from

    HTML is not a programming language; it is a markup language. This means
    structure only, no styling, no functionality, just structure.

    XHTML (a type of XML) which was a version of HTML that used
    stricter XML

    The goal was to have a more predictable and generalized format which allowed
    better rendering with less errors because browsers have to account for less

    HTML5 WHATWG↗ came up with a better version of HTML called HTML5 which was
    adopted by W3C in 2009
